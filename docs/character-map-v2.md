# 캐릭터 후보 매핑 v2

## 목적

추천 단위를 `캐릭터`로 제한합니다. 세부 플레이 방향은 후보로 만들지 않습니다.

캐릭터 안에서 플레이 방향이 여러 갈래로 나뉘더라도, 이 문서에서는 대표 플레이 방향 하나로만 평가합니다. 결과 페이지에도 세부 플레이 방향명은 표시하지 않습니다.

결과 화면에서는 캐릭터 설명 위에 해시태그를 표시합니다.

## 후보 벡터

벡터 범위는 `-1..+1`입니다.

| 축 | - 방향 | + 방향 |
| --- | --- | --- |
| `locus` | Outfield | Base |
| `style` | Direct | Setup |
| `target` | Self | Team |
| `difficulty` | Easy | Hard |

유형 코드는 `O/B + D/S + S/T + E/H`입니다. 이 코드는 사용자에게 노출하지 않고 URL path/state/debug 용도로만 사용합니다.

## 매칭 규칙

1. 일반 캐릭터는 사용자 벡터와 캐릭터 벡터의 거리로 정렬합니다.
2. Wonkey는 일반 순위에 섞지 않고 조건 충족 시 캐릭터 결과 형식으로 표시합니다.
3. 결과 페이지에는 캐릭터명, 대표 플레이 방향, 추천 이유, 주의점을 표시합니다. 유형 코드는 표시하지 않습니다.
4. 잘 어울리는 캐릭터를 함께 표시합니다.

거리 계산:

```text
distance =
  1.0 * abs(user.locus - character.locus) +
  1.0 * abs(user.style - character.style) +
  0.9 * abs(user.target - character.target) +
  1.2 * abs(user.difficulty - character.difficulty)
```

난도 축은 추천 실패 비용이 크므로 가중치를 약간 높게 둡니다.

- 잘 어울리는 캐릭터 2명

## 구현 데이터 형태

조합 데이터는 캐릭터 데이터와 분리해서 관리합니다.

```js
const characterRelations = {
  wolfgang: {
    synergies: [
      { id: "warly", reason: "전투 전 음식 버프로 보스전 성능을 크게 올립니다." },
      { id: "wortox", reason: "근접 전투 중 누적 피해를 빠르게 복구합니다." }
    ]
  }
};
```

조합 추천은 결과 설명 정보로 취급합니다. DLC 캐릭터도 구분 없이 그대로 표시합니다.

## 검토 필요 항목

- 캐릭터 벡터는 대표 플레이 방향 기준입니다. 세부 플레이 선택에 따른 차이는 의도적으로 반영하지 않습니다.
- `Direct / Setup`과 `Self / Team`은 일부 캐릭터에서 섞입니다. 결과 설명에서 혼합 성향을 명시해야 합니다.
- 조합 매트릭스는 초기 수동 규칙입니다. 실제 결과 페이지 검증 후 조정해야 합니다.
