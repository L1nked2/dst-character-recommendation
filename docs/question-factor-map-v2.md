# 질문-팩터 매핑 v2

## 목적

`questions.md`의 질문을 v2 추천 모델에 맞춰 재해석합니다. 기존 구현의 `beginner`, `combat`, `support` 같은 다중 weights는 비교용으로만 두고, 실제 추천 로직은 아래 4축 벡터를 기준으로 재작성합니다.

## 축 정의

벡터 범위는 `-2`부터 `+2`까지입니다.

| 축 | - 방향 | + 방향 |
| --- | --- | --- |
| `locus` | Outfield | Base |
| `style` | Direct | Setup |
| `target` | Self | Team |
| `difficulty` | Easy | Hard |

별도 보조 신호:

| 신호 | 의미 |
| --- | --- |
| `wnky` | Wonkey 와일드카드 추천 가능성 |

## 집계 규칙

1. Q1~Q12의 축 값을 합산합니다.
2. 각 축 합계를 `-1..+1`로 정규화합니다.
3. 후보 캐릭터 벡터와 거리 기반으로 매칭합니다.
4. 내부 상태용 유형 코드를 계산합니다.
5. `wnky` 합계가 일정 기준 이상이면 일반 추천과 별도로 Wonkey를 캐릭터 결과 형식으로 노출합니다.

초기 기준:

```text
wnky >= 5 이면 Wonkey 와일드카드 표시
wnky >= 7 이면 Wonkey를 와일드카드 1순위로 표시
```

## 구현 메모

현재 `script.js`의 질문 데이터는 이 문서의 v2 벡터를 기준으로 구성합니다. `weights`는 기획 비교용으로만 남기고 실제 추천 계산에는 사용하지 않습니다.
